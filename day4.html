Lecture Notes

In Day 4, the focus is on how to organize HTML files, use boilerplate code, and structure content using div, class, and id attributes.

  1. Introduction
    Overview of basic HTML concepts, file organization, and coding best practices.
  2. Understanding File Paths
    File paths specify the location of files on your computer or server.
  3. Types of File Paths
    There are two main types: relative paths and absolute paths.
  4. Relative Path
    Specifies a file location relative to the current file's directory.
  5. Code Example
    Example: <a href="folder/file.html">Link</a> uses a relative path to navigate.
  6. Absolute Path
    Specifies the full path from the root directory or server to the file.
  7. Note on Absolute Paths
    Absolute paths are fixed and work regardless of the current file location.
  8. Note on Relative Paths
    Relative paths are flexible and change depending on the current file location.
  9. Why Use Boilerplate Code
    Boilerplate code provides a standard structure to start a new HTML file quickly.
  10. Understanding Boilerplate Code
    Includes essential HTML elements like <!DOCTYPE html>, <html>, <head>, and <body>.
  11. UTF-8
    A character encoding standard that supports most characters from all languages.
  12. ASCII Values
    A character encoding system that represents letters, digits, and symbols using numbers 0‚Äì127.
  13. Why Use UTF-8
    UTF-8 ensures text displays correctly across different browsers and platforms.
  14. div tag
    The <div> tag is a container used to group elements for styling or layout.
  15. class & id
    Attributes used to uniquely identify (<id>) or group (<class>) elements for CSS or JS.
  16. Header , Main & Footer
    Semantic HTML tags that define the top, main content, and bottom sections of a page.
  17. Wrapping Up
    Concluding the lesson by summarizing key concepts about file paths, encoding, and HTML structure.
